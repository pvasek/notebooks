Internet — od kabelu ke stránce

Co se opravdu stane mezi kliknutím a tím, než se ti rozsvítí stránka? Pojďme to projít — od kabelu na dně oceánu až po zámeček v adresním řádku.

01 Internet a web nejsou totéž

Když řekneš „nejde mi internet", obvykle myslíš, že nejdou stránky. Jenže to není totéž. Internet je infrastruktura — propletená síť kabelů, routerů a datových center po celém světě. Mezi kontinenty vedou hlavně optické kabely po dně oceánu, doplněné satelity a mobilními vysílači.

Web (WWW) je jedna ze služeb, co po té síti běží. Vznikl v roce 1989 v CERNu, vymyslel ho Tim Berners-Lee. Funguje na principu „klikni na odkaz, otevře se stránka". Po těch stejných kabelech ale jezdí i e-maily, video hovory, online hry, cloudové zálohy a spousta dalších služeb.

Analogie

Internet je silnice. Po těch silnicích jezdí různá auta — osobní (web), nákladní (e-mail), záchranky (DNS), kamiony (cloud). Když nejede tramvaj, neznamená to, že nefungují silnice.

02 Cesta klikem ke stránce

Tohle je nejdůležitější obrázek celé lekce. Co se stane, když do prohlížeče napíšeš třeba cs.wikipedia.org/wiki/Praha a zmáčkneš Enter? Pojďme to projít krok po kroku — spusť animaci dole.

Klíčové role: prohlížeč (klient) chce stránku. DNS mu řekne, kam pro ni jít. Web server mu ji pošle. Cestou po internetu pakety prochází přes desítky routerů — krabic, které je směrují správným směrem (i tvůj domácí Wi-Fi router je router).

Pozor

Když dostane prohlížeč HTML, nemá ještě hotovo. HTML je jen kostra, uvnitř jsou odkazy na CSS soubory, obrázky a JavaScript. Pro každý prohlížeč pošle další požadavek. Jedna „stránka" se tak skládá z klidně desítek dotazů na server.

03 Adresy — IP, DNS, URL

IP adresa

Každé zařízení v síti má číselnou adresu. Stará verze IPv4 používá čtyři čísla 0–255 (např. 192.168.1.15) — adres je tak ~4,3 miliardy a už jich je málo. Nová IPv6 používá osm hexadecimálních skupin (2001:db8::1) — adres je řádově 10³⁸, prakticky neomezeně.

IP může být veřejná (vidí ji celý internet) nebo privátní (jen ve tvojí domácí síti — typicky 192.168.x.x). Více lidí v jedné domácnosti sdílí stejnou veřejnou IP.

DNS — telefonní seznam internetu

Pamatovat si IP adresy je nepříjemné. DNS (Domain Name System) ti to nemusí — přeloží srozumitelné jméno na IP. Když napíšeš seznam.cz, prohlížeč nejprve zavolá DNS a ten mu řekne: „je to 77.x.x.x". Teprve potom prohlížeč ví, kam dotaz poslat.

Otázka

Co by se stalo, kdyby DNS na celém světě přestalo fungovat? Stránky by se nedaly otevřít přes jména (`seznam.cz`), ale pokud bys znal IP, šly by načíst přímo přes http://77.x.x.x. Většina lidí ale IP nezná — internet by se tak prakticky zastavil.

URL (Uniform Resource Locator) je celá adresa, kterou píšeš do prohlížeče. Skládá se z několika částí, a tohle je důležité — protože phisheři to zneužívají. Pohraj si:

04 Z čeho se skládá stránka

Když server pošle prohlížeči HTML, jsou v něm odkazy na další soubory. Stránky jsou skoro vždy směs tří technologií:

  • HTML — kostra. Co je text, co je obrázek, co je nadpis.
  • CSS — vzhled. Barvy, písma, rozložení.
  • JavaScript — chování. Tlačítka, animace, formuláře, dnes i celé webové aplikace (e-maily v prohlížeči, mapy, hry).

Analogie

HTML je člověk. CSS je oblečení. JavaScript je co dělá.

Statický × dynamický web

Statický web = server pošle všem stejnou hotovou stránku. Třeba tahle lekce je statická — `.html` soubor leží na disku a každý ho dostane identický.

Dynamický web = server stránku skládá při každém požadavku z databáze. Wikipedie, e-shopy, vyhledávač, sociální sítě — pokaždé jiný obsah pro jiného uživatele. Backend (program běžící na serveru) si vytáhne data z databáze a sestaví HTML „na míru".

Z toho plynou tři role, o kterých dnes uslyšíš každého programátora:

  • Frontend — to, co vidíš v prohlížeči (HTML, CSS, JS).
  • Backend — to, co běží na serveru (databáze, logika).
  • API — způsob, jakým spolu frontend a backend mluví. Posílá si typicky strukturovaná data ve formátu JSON.

05 Bezpečnost

HTTPS — co opravdu zaručuje

Když má URL https:// a v adresním řádku je zámeček, prohlížeč ti říká: „přenos dat mezi tebou a tímhle serverem je šifrovaný." To znamená, že nikdo cestou nečte ani nemění data — soused na Wi-Fi, tvůj poskytovatel, kavárenský router. Dobré.

Pozor — velký mýtus

Zámeček neznamená, že webu můžeš věřit. Šifruje jen přenos, ale neříká, kdo je na druhé straně. Phisher si zaplatí HTTPS certifikát za půl hodiny zdarma a jeho falešná stránka bude mít zámeček úplně stejně jako pravá banka. Kontroluj doménu, ne zámeček.

Hesla — délka rozhoduje

Útočník dnes neháže do dveří jeden klíč za druhým — má počítač, který zkouší miliardy hesel za vteřinu. A začne těmi nejčastějšími slovy ze slovníku, ne náhodnými písmeny. Délka je proto víc než „složitost". Vyzkoušej si:

Pamatuj

Dlouhé heslo + správce hesel + dvoufaktor (2FA). To je celá tajná receptura na 90 % bezpečnosti běžného uživatele. Správce hesel (1Password, Bitwarden) ti vygeneruje a zapamatuje unikátní heslo pro každý web. 2FA znamená, že kromě hesla potřebuješ ještě druhý krok (kód z aplikace, klíč) — i když ti někdo heslo ukradne, bez toho druhého se nikam nedostane.

Phishing — kdo to je?

Phishing je nejčastější útok na běžného uživatele. Útočník nepotřebuje umět hacknout banku — stačí ho přesvědčit tebe, abys mu sám napsal heslo. Falešná stránka, falešný e-mail, falešná SMS. Vodítka: tlak na rychlost („do 24 hodin!"), divná URL, nečekané přílohy, citově laděné výzvy.

Cookies a digitální stopa

Cookies jsou drobné kousky dat, které si web ukládá do tvého prohlížeče. Některé jsou nezbytné — drží přihlášení, košík v e-shopu, nastavený jazyk. Jiné jsou sledovací — předávají si informace mezi weby, aby ti reklamní systémy mohly cílit reklamu. Proto v EU musí weby ptát na souhlas.

Digitální stopa = vše, co po sobě na internetu necháváš: příspěvky, lajky, komentáře, historie hledání, metadata fotek (často včetně GPS). Internet má dlouhou paměť — i „smazaný" post může být uložený jako screenshot, v cache vyhledávače nebo v archivu (Wayback Machine). Než něco vystavíš, představ si, že to po deseti letech čte tvůj zaměstnavatel.

★ Vyzkoušej si to

Co si odnést

  1. Internet = infrastruktura (kabely, routery, IP). Web = jedna ze služeb nad ní.
  2. Když klikneš: prohlížeč zeptá DNS na IP, pošle požadavek serveru, dostane HTML, potom si stáhne CSS/JS/obrázky a teprve pak vykreslí.
  3. V URL je nejdůležitější doména + TLD. Subdoménu si může pojmenovat kdokoli.
  4. HTTPS šifruje přenos. Neříká ti, jestli serveru můžeš věřit.
  5. Dlouhé heslo + správce hesel + 2FA. A u každého e-mailu se ptej, jestli odesílatel a URL skutečně sedí.
